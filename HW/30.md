#### Пример 1

Метод обработки связей сущности "Телепередача"(программа). Есть три относительно автономных блока, каждый из которых отвечает за создание того или иного вида связи.

```java
private void processLinks(ExecutionContext ctx, TVGridType.Event event, int actionType, long programId) throws Exception {
    // Удаляем старые связи программы с категориями, жанрами и тегами
    if (actionType == UPDATE_EPG_ACTION) {
        CommonDao.delChildFromObjectLink(ctx.getSession(), "PROGRAMINPROGRAMCATEGORY", programId);
        CommonDao.delChildFromObjectLink(ctx.getSession(), "PROGRAMINPROGRAMCONTENTGENRE", programId);
        CommonDao.delChildFromObjectLink(ctx.getSession(), "CONTENTINTAG", programId);
    }

    // Добавляем категории к программе
    TVGridType.Event.Categories categories = event.getCategories();
    if (categories != null) {
        for (TVGridType.Event.Categories.Category category : categories.getCategoryArray()) {
            Long objId = getProgramCategoryId(ctx, category);
            if (objId != null) {
                CommonDao.mergeIntoObjectLink(ctx.getSession(), "PROGRAMINPROGRAMCATEGORY", objId, programId);
            }
        }
    }

    //Добавляем жанры к программе
    TVGridType.Event.Genres genres = event.getGenres();
    if (genres != null) {
        for (TVGridType.Event.Genres.Genre genre : genres.getGenreArray()) {
            Long programContentGenreId = getProgramContentGenreId(ctx, genre);
            if (programContentGenreId != null) {
                CommonDao.mergeIntoObjectLink(ctx.getSession(), "PROGRAMINPROGRAMCONTENTGENRE",
                        programContentGenreId, programId);
            }
        }
    }

    //Добавляем теги к программе
    TVGridType.Event.Tags tags = event.getTags();
    if (tags != null) {
        for (TVGridType.Event.Tags.Tag tag : tags.getTagArray()) {
            Long tagId = getTag(ctx, tag);
            if (tagId != null) {
                CommonDao.mergeIntoObjectLink(ctx.getSession(), "CONTENTINTAG", tagId, programId);
            }
        }
    }
}
```

private void processChannel(ExecutionContext ctx, Channel channel) throws Exception {
    Set<String> curIdSet = getExternalIdsOfPublishedEpg(ctx, channel);

    Set<String> curIdSetForSingleImport = curIdSet;
    if ((isSingleImport() && !isMergeModeImport()) || isForceImport()) {
        curIdSet = new HashSet<>();
    }

    long t1 = System.currentTimeMillis();

    TVGridType tvGrid = getTvGridType(channel);
    TVGridType.Event[] events = tvGrid.getEventArray();

    long t2 = System.currentTimeMillis();
    timeLoadXml = timeLoadXml + (t2 - t1);
    removeCount = 0;
    updateCount = 0;
    addCount = 0;

    minStartDateScheduleId = null;
    minStartDate = null;
    maxStartDateScheduleId = null;
    maxStartDate = null;

    for (TVGridType.Event event : events) {
        processEvent(ctx, channel, event, curIdSet);
    }

    if ((isSingleImport() && !isMergeModeImport()) || isForceImport()) {
        curIdSet = curIdSetForSingleImport;
    }
    removeCount = curIdSet.size();

    // ---------------------------------------------------------------------------
    // Распубликация(софт делит) старых программ передач. Автономный блок.
    // ---------------------------------------------------------------------------
    // в этом сете у нас остались только те ID которые надо распубликовать
    if (curIdSet.size() > 0) {
        Collection<ParameterBinding> bindings = set(
                bind("SS", channel.getId()),
                bind("LOC", channel.getLocationId()),
                bind("SECTION", channel.getSection()),
                new StringParameterBinding("DISCRIMINATOR", getEpgDiscrByType(channel.getEpgType())),
                new DateTimeParameterBinding("PROCESS_EPG_FROM_DATE", channel.getProcessEpgFromDate()),
                new DateTimeParameterBinding("PROCESS_EPG_TO_DATE", channel.getProcessEpgToDate())
        );

        t1 = System.currentTimeMillis();

        StringBuilder updatePart = new StringBuilder(
                "UPDATE SCHEDULE S\n" +
                "   SET S.IS_PUBLISHED = 3 \n"
        );
        StringBuilder wherePart = new StringBuilder(
                GET_SCHEDULE_FOR_PARAMS_WHERE_PART +
                "   AND ("
        ).append(DbCommand.getLongInClauseStr("S.EXTERNALID", curIdSet, bindings, "EID")).append(" )");

        // затем распубликовываем ненужные программы передач
        StringBuilder updateSql = new StringBuilder().append(updatePart).append(wherePart);
        ctx.getSession().executeUpdate(updateSql.toString(), bindings);
        //-------------------------------------------------

        t2 = System.currentTimeMillis();
        timeDbUpdate = timeDbUpdate + (t2 - t1);
    }
}

#### Пример 3

Метод для оформления временной покупки контента. Тут есть блок кода, который проверяет не была ли уже оформлена эта покупка.

```java
private void temporaryPurchase(PurchaseParams params) {
        session.doInTransaction(() -> {
            //-----------------------------
            // Проверка статуса контента(уже подключен или ещё нет). Автономный блок.
            //-----------------------------
            String accountNumber = params.getAccount().getAccountNumber();
            Optional<PurchaseDto> oldPurchase = purchaseDao.findBy(accountNumber,
                    params.getContentId(),
                    params.getContentPackage().getId(),
                    params.getContentUsageRule().getId());

            // Если контент есть и уже подключен, ничего не делаем.
            if (oldPurchase.isPresent()
                && oldPurchase.get().getStatus() == Status.CONFIRMED
                && oldPurchase.get().getEndDate().isAfter(LocalDateTime.now())) {
                return;
            }
            //----------------------------------
            
            LocalDateTime startDate = LocalDateTime.now();
            LocalDateTime tempEndDate = getTemporaryEndDate(startDate);

            Purchase purchase = toPurchaseConverter.convert(params).toBuilder()
                    .status(TEMPORARY)
                    .startDate(startDate)
                    .endDate(tempEndDate)
                    .build();
            // Обновим статус покупки, если есть, или запишем покупку
            long purchaseId;
            if (oldPurchase.isPresent()) {
                purchaseId = oldPurchase.get().getId();
                purchaseDao.update(purchaseDtoToPurchaseConverter.convert(oldPurchase.get())
                        .toBuilder()
                        .status(purchase.getStatus())
                        .startDate(purchase.getStartDate())
                        .endDate(purchase.getEndDate())
                        .price(purchase.getPrice())
                        .usageDetails(null)
                        .build());
            } else {
                purchaseId = purchaseDao.create(purchase);
            }

            // Делаем запись в историю покупок.
            PurchaseHistory.PurchaseHistoryBuilder purchaseHistoryBuilder = toPurchaseHistoryConverter.convert(params).toBuilder()
                    .purchaseId(purchaseId)
                    .startDate(startDate)
                    .endDate(tempEndDate)
                    .wsResult("success")
                    .status(Status.TEMPORARY);

            if (params.getSourceType() == BANK_CARD) {
                purchaseHistoryBuilder.wsMethod(".../purchase_bank_card_temporary");
                purchaseHistoryBuilder.paymentId(params.getPaymentId());
            }
            purchaseHistoryDao.create(purchaseHistoryBuilder.build());
            
            serviceAccountInfoDao.updatePurchaseDate(accountNumber);
            messageService.sendPurchasesChangedNotification(accountNumber);
        });
    }
```

#### Пример 4

Метод для замены контента. Тут есть блок кода, в котором собирается информация для обмена. Она используется для принятия решения об обмене и его оформления.

```java
public void contentExchange(ExchangeParams rawParams) {
        session.doInTransaction(() -> {
            PurchaseParams params = exchangeParamsConverter.convert(rawParams);
            purchaseExistsValidator.validate(params);
            Long migrationId = null;
            if (ConfigurationHelper.getBooleanProperty("new-purchases.support-old-scheme", true)) {
                migrationId = purchaseService.exchangeContent(params);
            }

            //---------------------------
            // Получаем информацию необходимую для обмена контента. Автономный блок.
            //---------------------------
            PurchaseDto exchangingContent = purchaseDao.findAllBySanAndFilters(params.getAccount().getAccountNumber(),
                            null, params.getExchangeContentId(), null, CONFIRMED, null, true)
                    .stream()
                    // Берём последнюю запись о покупке контента, который будем менять.
                    .max(Comparator.comparing(PurchaseDto::getCreatedAt))
                    .orElseThrow(() -> new RuntimeException("Не найдено подходящего контента для обмена"));

            long originPurchaseId = wasExchange(exchangingContent)
                    ? exchangingContent.getUsageDetails().getExchange().getOriginPurchaseId()
                    : exchangingContent.getId();

            long exchangeCount = wasExchange(exchangingContent)
                    ? exchangingContent.getUsageDetails().getExchange().getCount() + 1
                    : 1;
            //--------------------------

            // Проверим, что не достигнут лимит обменов
            if (exchangeCount - 1 >= ConfigurationHelper.getIntProperty("purchase_content.exchange_limit", 10)) {
                throw new PurchaseExchangesLimitReachedException();
            }

            // Если требуемый контент уже подключался, то нужно обновить соответствующую запись, а не создавать новую.
            Optional<PurchaseDto> oldPurchase = purchaseDao.findBy(params.getAccount().getAccountNumber(),
                    params.getContentId(),
                    params.getContentPackage().getId(),
                    params.getContentUsageRule().getId());

            Purchase purchase = toPurchaseConverter.convert(params).toBuilder()
                    .usageDetails(UsageDetails.builder()
                            .type(UsageDetails.Type.EXCHANGE)
                            .exchange(Exchange.builder()
                                    .originPurchaseId(originPurchaseId)
                                    .parentPurchaseId(exchangingContent.getId())
                                    .count(exchangeCount)
                                    .build())
                            .build())
                    .migrationId(migrationId)
                    .build();

            // Обновим статус покупки, если она есть, или запишем новую покупку
            long purchaseId;
            if (oldPurchase.isPresent()) {
                purchaseId = oldPurchase.get().getId();
                purchaseDao.update(purchaseDtoToPurchaseConverter.convert(oldPurchase.get())
                        .toBuilder()
                        .status(purchase.getStatus())
                        .startDate(purchase.getStartDate())
                        .endDate(purchase.getEndDate())
                        .price(purchase.getPrice())
                        .usageDetails(purchase.getUsageDetails())
                        .migrationId(purchase.getMigrationId())
                        .build());
            } else {
                purchaseId = purchaseDao.create(purchase);
            }

            // Обновим статус контента, который заменили
            purchaseDao.update(purchaseDtoToPurchaseConverter.convert(exchangingContent).toBuilder()
                    .status(EXCHANGED)
                    .build());

            PurchaseHistoryDto originPurchaseHistory = purchaseHistoryDao.getActiveByPurchaseIdUsageTypeAndStatus(
                    originPurchaseId, PURCHASE, CONFIRMED);

            // Сделаем запись в историю покупок.
            purchaseHistoryDao.create(toPurchaseHistoryConverter.convert(params).toBuilder()
                    .purchaseId(purchaseId)
                    .parentId(originPurchaseHistory.getId())
                    .wsResult("success")
                    .status(CONFIRMED)
                    .usageDetails(purchase.getUsageDetails())
                    .migrationId(migrationId)
                    .build());

            messageService.sendPurchasesChangedNotification(params.getAccount().getAccountNumber());
        });
    }
```

#### Пример 5

Метод по обогащению записей об услугах разными данными. Тут есть несколько блоков кода, в рамках которых собираются разные справочники. Затем эти справочники применяются для заполнения сведений об услуге.  

```java
private Collection<ServiceDto> enrichServices(Collection<ServiceDto> allServices, 
                                              Collection<OptionalService> servicesOnAccount, 
                                              String accountNumber, 
                                              Source.Type sourceType) {

    final Map<Long, ServiceDto> ssIdToService = servicesOnAccount.stream()
            .map(OptionalService::getService)
            .filter(Objects::nonNull)
            .collect(Collectors.toMap(ServiceDto::getServiceSpecId, s -> s));
    
    //---------------------------
    // Получаем справочник бонусных программ. Автономный блок
    //---------------------------
    List<Long> bpIds = servicesOnAccount.stream()
            .map(os -> os.getService().getBonusProgramSpecId())
            .filter(Objects::nonNull)
            .distinct()
            .collect(Collectors.toList());
    Map<Long, BonusProgramSpecDto> bonusProgramIdToBonusProgram = bonusProgramSpecDao.findByIds(bpIds).stream()
            .filter(Objects::nonNull)
            .collect(Collectors.toMap(BonusProgramSpecDto::getId, bp -> bp));
    //---------------------------
    
    Map<Long, CurrencyDictDto> currencyIdToCurrency = currencyDao.findAll().stream()
            .collect(Collectors.toMap(CurrencyDictDto::id, c -> c));

    //---------------------------
    // Справочник внешних биллингов. Автономный блок
    //---------------------------
    List<Long> extBillingIds = servicesOnAccount.stream()
            .map(os -> os.getService().getExternalBillingId())
            .filter(Objects::nonNull)
            .distinct()
            .collect(Collectors.toList());

    Map<Long, ExtBillingDto> extBillingIdToExtBilling = externalBillingDao.findByIds(extBillingIds).stream()
            .filter(Objects::nonNull)
            .collect(Collectors.toMap(ExtBillingDto::getId, b -> b));
    //---------------------------

    for (ServiceDto serviceDto : allServices) {
        ServiceDto service = ssIdToService.get(serviceDto.getId());
        if (serviceDto.isOptional() && service != null) {
            serviceDto.setParentId(service.getParentId());
            serviceDto.setStandaloneSubscribe(service.isStandaloneSubscribe());
            serviceDto.setStartDate(service.getStartDate());
            serviceDto.setStopDate(service.getStopDate());
            serviceDto.setState(service.getServiceState() == null
                    ? ServiceState.NOT_ACTIVE
                    : service.getServiceState());
            serviceDto.setSource(getSource(service, bonusProgramIdToBonusProgram, extBillingIdToExtBilling));
            Integer price = getPrice(service);
            serviceDto.setPrice(price);
            CurrencyDto currency = getCurrency(price, service, bonusProgramIdToBonusProgram, currencyIdToCurrency);
            serviceDto.setCurrency(currency);
        }
    }

    if (sourceType == null || sourceType == PROMOACTION) {
        List<PromoService> promoServices = promoServiceDao.getActivePromoServices(accountNumber);
        // мержим опциональные услуги с промо
        enrichWithPromoServices(allServices, promoServices);
    }

    return allServices;
}
```

#### Вывод
Насколько я понял, код считается лапшой, когда в нём выполняются разные по смыслу вещи. Важно учиться видеть потоки данных в программе, 
подниматься над уровнем кода и думать в категориях данных. Что мы представляем в коде, как эти данные связаны и т.д. 
Если мы будем видеть потоки обработки данных, то сможем структурировать код более простым для восприятия образом. А это повысит его сопровождаемость. 
Также мы сможем увидеть раздутость кода и предпринять какие-то действия.
Понравилась цитата Торвальдса из материала:
> Плохие программисты думают о коде. Хорошие программисты думают о структурах данных и взаимосвязях между ними

Недавно разбирал большой кусок легаси-кода и вспоминал эту цитату. Код сначала казался страшным и очень запутанным. Но потом я постарался подняться немного выше 
и сосредоточить своё внимание на данных и том, что с ними происходит в целом. После этого всё стало гораздо проще, я увидел, как и куда перемещаются данные, понял зачем это делается.
И как будто бы разглядел даже некоторую красоту в этом куске легаси, потому что на уровне данных вся картина воспринимается проще и ты начинаешь понимать, что здесь приняли неплохие решения, с учётом стоявших проблем.
Цитата Торвальдса меня зацепила, поскольку она напоминает о том, что важно на самом деле. Буду стараться чаще смотреть на проект в первую очередь с точки зрения данных и их связей. 

По ходу выполнения задания несколько раз задавался вопросом "Как понять, что кусок кода излишне выделять в функцию?". Где та граница, после которой код нужно выделить в функцию? 
Код делает слишком мало или нужен лишь в одном месте? Нужно ли всегда выделять функцию, если есть возможность? 
Кажется, что нет. Тут нужно смотреть на конкретную ситуацию и чрезмерное увлечение выделением функций может путать, так как постоянно придётся прыгать между ними, чтобы понять, как что-то делается.
С другой стороны, если в коде много функций с ясными именами, то по ним и не нужно прыгать, чтобы понять, что происходит с данными. 
В общем, тут ещё есть над чем подумать.
