У нас имелся метод, который выполняет GET-запрос к API определённой системы и возвращает ответ в виде массива байтов. В целом, массив байтов довольно универсальная штука, но всё же это конкретный объект с определёнными свойствами.

**Стало:**
```java
private <T> T get(String path, Class<T> type) {
    HttpGet httpGet = new HttpGet(getSecretSystemApiUrl(path));

    return request(httpGet, type);
}

private <T> T post(String path, Object body, Class<T> type) {
    HttpPost httpPost = new HttpPost(getSecretSystemApiUrl(path));
    setEntity(httpPost, body);

    return request(httpPost, type);
}

private <T> T request(HttpUriRequest request, Class<T> type) {
    try (CloseableHttpClient client = httpClientFactory.get();
        CloseableHttpResponse response = client.execute(request)) {
        HttpEntity entity = response.getEntity();
        if (isSuccess(response)) {
            return getResponse(entity, type);
        }

        throw SecretSystemException(entity);
    } catch (IOException e) {
        LOGGER.error(e.getMessage(), e);

        throw new NetworkException(String.format("Can not %s '%s'", request.getMethod(), request.getURI()), e);
    }
}
```

Продолжаем повышать полиморфизм наших http-методов из прошлого примера. Раньше для каждого типа http-запроса приходилось описывать свою логику, что приводило к дублированию кода, увеличению количества методов и т.д.
В новой версии, общая часть выделена в универсальный полиморфный метод `request`, который способен работать с любым http-методом. Новый метод не знает о типе запроса, который ему передадут. Также он не знает и о типе ответа, который он должен вернуть.
Это опять накладывает определённые ограничения на его реализацию, а также позволяет нам протестировать код единожды и потом спокойно его переиспользовать в разных ситуациях.  
Для удобства можно завести методы-обёртки с популярными типами запросов(get, post и т.д.).

    private final AccountEnvelopeChangeValidator validator;
    private final MwapiService mwapiService;

    @Override
    protected CommandResult invokeWithTransactionEvent(ExecutionContext ctx, Map<String, String> rawParams) {
