### Принципы поведения системы
**Создание объектов:**  
Большая часть объектов, связанных с игрой, будет инициализироваться в классе `GameController`, который отвечает за управление игрой(паттерн посредник).
За создание элементов игрового поля будет отвечать фабрика `ElementsFactory`.

**Обработка событий:**  
Обработка основных игровых событий, будет происходить с помощью паттерна наблюдатель:
- Начисление очков счёта
- Отрисовка изменений на игровом поле
- Сохранение истории ходов

Обработку пользовательского ввода будет осуществлять класс `GameController`. Он будет валидировать корректность введённых координат и команд,
в случае нарушений будет выводиться сообщение об ошибке и предложение ввести команду/координаты повторно.
Игрок будет иметь возможность:
- Запустить игру командой `Старт`
- Перезапустить игру командой `Перезапуск`
- Закончить игру командой `Стоп`.

При завершении игры будет показана статистика: количество сделанных ходов и заработанных очков.

**Общий цикл игры:**  
Подготовка:   
1. Запуск с указанием способа отображения игрового поля(показывать каждое обновление или итоговый результат цикла).
2. Инициализация игрового поля и статистики.
3. Заполнение поля случайными элементами: `fillGaps()`.
4. Проверка наличия комбинаций `hasCombinations()` и их удаление `processCombinations()`.  

Шаги 3 и 4 повторяются до тех пор, пока поле встречаются комбинации.
В итоге получаем заполненное игровое поле, без комбинаций. 

Основной цикл:  
1. Проверка наличия возможных ходов `hasPossibleMoves()`.
2. Обработка пользовательского ввода, попытка перестановки элементов.
3. Проверка наличия комбинаций `hasCombinations()` и их удаление `processCombinations()`.
4. Обновление статистики(запомнили ход и/или начислили очки).
5. Заполнение поля случайными элементами: `fillGaps()`.
6. Вывод состояния игрового поля в консоль. В режиме последовательных шагов вывод происходит после каждого изменения состояния поля, а не только в конце цикла.

Шаги 3-5 повторяются до тех пор, пока мы не получим заполненное игровое поле, без комбинаций.
Основной цикл повторяется до тех пор, пока есть возможные ходы или игрок не введёт команду для завершения игры. Затем выполняется вывод статистики и программа завершает работу.

**Тесты для типовых сценариев:**  
1. Поле заполняется случайными элементами.
2. Игрок не может просто так переставить элементы(должна появиться комбинация, либо сработать специальный эффект бонусного элемента).
3. Комбинации обрабатываются, игрок получает за неё очки, ход сохраняется в истории. 
4. За составление сложных комбинаций на поле появляются бонусные элементы.
5. При перемещении бонусного элемента срабатывает его специальный эффект.
6. Корректное заполнение пропусков на поле: сначала верхние элементы опускаются вниз, затем появляются новые.
7. Шаги 3-5 основного цикла выполняются до тех пор, пока мы не получим заполненное игровое поле, без комбинаций.
8. Обработка некорректного ввода. Игра не ломается при вводе некорректной команды/координаты.
9. Игровой цикл завершается, когда на поле закончились возможные ходы.
