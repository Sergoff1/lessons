Допустим у нас есть иерархия услуг, которые мы предоставляем клиентам по подписке.
Пускай в иерархии будут классы **Услуга** и **Промо услуга**. Унаследуем Промо услугу от обычной Услуги.
У них есть метод "подключить", который делает услугу доступной для пользователя.

Предусловие в классе **Услуга** для метода "подключить" может быть таким: Услуга есть в тарифном плане пользователя и не подключена.  
В классе **Промо услуга** для этого же метода может быть такое предусловие: Услуга есть в тарифном плане пользователя и подключается впервые.  
Здесь мы видим усиление предусловия. А значит при подключении коллекции услуг, мы можем нарваться на Промо услугу и получить ошибку, если такая услуга ранее 
подключалась пользователем и ему больше недоступен промо период. Это нарушает LSP.

Постусловие в классе **Услуга** для метода "подключить" может быть таким: Услуга становится доступна пользователю на **месяц**.  
В классе **Промо услуга** для этого же метода может быть такое постусловие: Услуга становится доступна пользователю на **неделю**.  
Здесь постусловие ослабляется и ломает логику родителя, тем самым опять нарушая LSP.
