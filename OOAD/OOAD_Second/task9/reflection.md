Суть я уловил и, в общем, задание сделал верно.
Использовал обобщения, там где они нужны. Верно уловил идею о том, что clone и copy похожи и для их реализации можно использовать один код. 
И даже использовал хак с готовым маппером для сериализации/десериализации.

Но есть ряд недочётов.  
Неверно задал входной параметр для метода equals. В эталоне -- Object, а у меня -- General. При этом реализация метода совпадает с эталонной.  
Добавил параметр в метод сериализации, хотя он там не нужен. В моём случае для выполнения сериализации нужно вызвать метод объекта и в качестве параметра передать этот же объект.
Это лишнее и параметр тут не нужен. Реализация же, совпадает с эталонной.  
Мой метод десериализации менее гибкий. В эталонном методе, можно задать тип, в который нужно десериализовать входную строку. Это позволяет использовать метод для десериализации любых объектов.
В моём же решении, в качестве параметра, принимается лишь строка для десериализации, а тип вычисляется на основе объекта, у которого мы вызываем метод десериализации.
