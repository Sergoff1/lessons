Если мы смотрим на модуль как на синтаксическую единицу, а не как класс, то тут ближайшим аналогом в Java будет скорее всего пакет.  
Получается, в Java удовлетворяются принципы 2, 3 и 5.

1. Модуль нельзя параметризовать типом.
2. Можно объединить в один модуль(пакет) несколько функций(классов), которые общаются друг с другом.
3. Модуль может входить в семейство других пакетов, направленных на решение одной задачи. У нас может быть пакет, который включает в себя другие пакеты/классы служащие для решения одной задачи.
4. В Java нет механизма наследования модулей, поэтому нет возможности динамически выбирать реализацию модуля.
5. У нас может быть пакет, который включает в себя другие модули/классы, выполняющие одну задачу, но отличающиеся лишь деталями реализации. Например, модуль генератора отчётов, содержит модули с разными реализациями генераторов(html, pdf).
